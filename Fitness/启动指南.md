# 健身管理系统启动指南

## 概述

本指南提供了健身管理系统的启动和停止方法。系统包含前端Vue应用和后端Spring Boot应用。

## 系统要求

- Java 17 或更高版本
- Maven 3.6 或更高版本
- Node.js 16 或更高版本
- npm 8 或更高版本
- PowerShell（Windows系统）

## 启动方法

### 方法一：使用批处理启动器（推荐）

1. 双击运行 `启动器.bat`
2. 选择选项 1 启动系统
3. 选择选项 2 停止系统

### 方法二：使用PowerShell脚本

#### 启动系统

1. 右键点击 `start-fitness-system.ps1`
2. 选择 "使用PowerShell运行"
3. 或在PowerShell中执行：
   ```powershell
   Set-Location D:\AFitness\Fitness
   .\start-fitness-system.ps1
   ```

#### 停止系统

1. 右键点击 `stop-fitness-system.ps1`
2. 选择 "使用PowerShell运行"
3. 或在PowerShell中执行：
   ```powershell
   Set-Location D:\AFitness\Fitness
   .\stop-fitness-system.ps1
   ```

### 方法三：手动启动

#### 启动后端服务

```powershell
cd D:\AFitness\Fitness
mvn spring-boot:run
```

#### 启动前端服务

```powershell
cd D:\AFitness\Fitness\frontend
npm run dev
```

## 访问地址

- 前端应用: http://localhost:3001
- 后端API: http://localhost:8080/api/v1

## 故障排除

### 端口占用问题

如果遇到端口占用问题，可以：

1. 运行 `stop-fitness-system.ps1` 停止所有服务
2. 手动终止占用端口的进程：
   ```powershell
   # 查看端口占用
   netstat -ano | findstr :8080
   netstat -ano | findstr :3001
   
   # 终止进程（将PID替换为实际的进程ID）
   taskkill /PID <PID> /F
   ```

### PowerShell执行策略问题

如果PowerShell提示无法执行脚本，可以更改执行策略：

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 依赖安装问题

确保所有依赖已正确安装：

1. 检查Java和Maven：
   ```powershell
   java -version
   mvn -version
   ```

2. 检查Node.js和npm：
   ```powershell
   node -v
   npm -v
   ```

3. 安装前端依赖：
   ```powershell
   cd D:\AFitness\Fitness\frontend
   npm install
   ```

4. 安装后端依赖：
   ```powershell
   cd D:\AFitness\Fitness
   mvn clean install
   ```

## 日志查看

启动脚本会自动显示服务日志。如需单独查看日志：

- 后端日志：在启动后端的PowerShell窗口中查看
- 前端日志：在启动前端的PowerShell窗口中查看

## 注意事项

1. 首次启动可能需要较长时间，因为需要下载依赖和编译代码
2. 启动顺序很重要：先启动后端服务，再启动前端服务
3. 停止系统时，请使用提供的停止脚本，确保所有进程都被正确终止
4. 如果修改了代码，前端会自动热更新，后端需要重启
