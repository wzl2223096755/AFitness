# 基于 Vue3 + Spring Boot 的智能健身管理系统设计与实现

## 摘要
随着生活水平的提高和健康意识的增强，科学健身已成为大众关注的热点。本文设计并实现了一个基于前后端分离架构的智能健身管理系统。前端采用 Vue3 + Vant UI + Pinia，后端采用 Spring Boot + MySQL + JWT 安全框架。系统实现了用户管理、训练计划定制、健身数据记录、1RM 计算分析以及训练负荷监控等功能。通过图表可视化展示，用户可以直观了解自己的运动表现和身体状态，从而制定更科学的训练计划。

**关键词**：Vue3；Spring Boot；健身管理；1RM 计算；前后端分离

---

## 1. 绪论

### 1.1 选题背景
在快节奏的现代生活中，健身已成为人们维持健康、释放压力的重要手段。然而，许多初学者由于缺乏科学的指导，往往难以制定合适的训练计划，或者无法准确记录和分析自己的训练进展。传统的纸笔记录方式效率低下，难以进行长期的趋势分析。

### 1.2 选题意义
开发一套智能健身管理系统，能够帮助用户：
1. **科学量化训练**：通过 1RM 计算器和容量分析，科学设定训练负荷。
2. **数据驱动决策**：通过可视化图表观察训练趋势，及时调整计划。
3. **提高训练效率**：系统化管理训练动作和计划，减少训练盲目性。

## 2. 系统关键技术介绍

### 2.1 前端技术栈
- **Vue 3**: 渐进式 JavaScript 框架，采用 Composition API 提升代码复用性。
- **Vant UI / Element Plus**: 提供丰富的移动端和 PC 端组件，加速界面开发。
- **Pinia**: 现代化状态管理库，替代 Vuex。
- **ECharts**: 强大的可视化图表库，用于数据统计。

### 2.2 后端技术栈
- **Spring Boot**: 简化 Spring 应用开发的框架，提供开箱即用的特性。
- **Spring Data JPA**: 简化数据库访问层代码，支持多种数据库。
- **MySQL**: 关系型数据库，存储用户信息和训练数据。
- **JWT (JSON Web Token)**: 实现无状态认证，保障接口安全性。

## 3. 系统需求分析

### 3.1 业务流程分析
1. **注册登录**：用户通过手机号/邮箱注册，登录后获取身份令牌。
2. **计划定制**：用户根据自身目标（增肌/减脂）选择或创建训练计划。
3. **数据打卡**：每次训练后记录动作、重量、组数、次数。
4. **统计分析**：系统自动计算训练容量、1RM，并生成趋势图。

### 3.2 功能需求
- **用户管理**：注册、登录、个人信息修改、密码重置。
- **训练计划管理**：创建周计划、添加动作、标记完成。
- **健身数据记录**：记录抗阻训练、有氧运动数据。
- **1RM 计算器**：支持多种经典公式估算极限力量。
- **统计报表**：训练量趋势、动作分布、恢复状态评估。

## 4. 系统架构设计

### 4.1 技术架构设计
系统采用标准的前后端分离架构（B/S模式）：
- **表现层**：Vue 3 负责页面渲染和交互。
- **接口层**：RESTful API 作为前后端沟通桥梁。
- **业务逻辑层**：Spring Boot 处理核心业务，如 1RM 计算算法、计划生成逻辑。
- **数据访问层**：Spring Data JPA 负责与 MySQL 交互。

### 4.2 功能模块设计
1. **认证模块**：SecurityConfig + JWT 实现安全控制。
2. **数据模块**：FitnessDataController 处理基础记录的增删改查。
3. **分析模块** : LoadRecoveryController 负责负荷计算和建议生成。
4. **报表模块**：DashboardController 聚合多维度数据供前端展示。

## 6. 系统实现

### 6.1 核心功能实现代码
#### 6.1.1 1RM 计算算法 (Java)
```java
public Double calculateOneRepMax(Double weight, Integer reps, String model) {
    switch (model) {
        case "Brzycki":
            return weight / (1.0278 - 0.0278 * reps);
        case "Lombardi":
            return weight * Math.pow(reps, 0.1);
        case "OConner":
            return weight * (1 + 0.025 * reps);
        case "Epley":
        default:
            return weight * (1 + (double) reps / 30);
    }
}
```
