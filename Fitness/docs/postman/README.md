# AFitness Postman 集合

本目录包含 AFitness 健身管理系统的 Postman API 集合文件。

## 文件说明

- `AFitness_API_Collection.json` - 完整的 API 接口集合

## 使用方法

### 1. 导入集合

1. 打开 Postman
2. 点击 "Import" 按钮
3. 选择 `AFitness_API_Collection.json` 文件
4. 点击 "Import" 完成导入

### 2. 配置环境变量

集合中已包含以下变量：

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `baseUrl` | `http://localhost:8080` | API 基础地址 |
| `accessToken` | (空) | JWT 访问令牌 |
| `refreshToken` | (空) | JWT 刷新令牌 |

如需修改 `baseUrl`，可以在集合变量中进行设置。

### 3. 获取认证令牌

1. 首先调用 "认证管理" 文件夹下的 "用户登录" 接口
2. 登录成功后，`accessToken` 和 `refreshToken` 会自动保存到集合变量中
3. 后续请求会自动使用保存的 `accessToken` 进行认证

### 4. 接口分类

集合中的接口按功能分为以下几类：

- **认证管理** - 登录、注册、令牌刷新等
- **训练管理** - 训练记录 CRUD
- **负荷恢复** - 1RM 计算、恢复评估
- **营养记录** - 营养数据管理
- **仪表盘** - 数据概览和分析
- **用户管理** - 用户资料和设置
- **训练计划** - 训练计划管理
- **健康检查** - 系统健康状态
- **管理员接口** - 系统监控和管理（需要管理员权限）

## 注意事项

1. 大部分接口需要认证，请先登录获取令牌
2. 管理员接口需要使用管理员账号登录
3. 日期格式统一使用 `yyyy-MM-dd`
4. 所有请求和响应均使用 JSON 格式
